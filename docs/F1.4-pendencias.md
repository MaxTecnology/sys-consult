# F1.4 – Alertas de Mensagens Críticas (Resumo)

## O que foi feito
- Widget `DteAlertsWidget` no dashboard: contadores de não lidas, críticas (`requere_atencao` + status interno `novo`), certificados vencendo e automações com erro.
- Comando `dte:alert-critical` envia e-mail usando destinatários das empresas das mensagens + `DTE_ALERT_EMAILS`; agendado hourly em `routes/console.php`.
- Critérios atuais do comando: `requere_atencao = true` **ou** `lida_sefaz = false` com `data_envio <= now() - since-hours` (padrão 6h). Limite padrão: 50 registros.

## Como usar/testar
```bash
# configurar destinatários (csv)
# opcional: admin copy
echo "DTE_ALERT_EMAILS=ops@exemplo.com,contato@exemplo.com" >> .env

# rodar manualmente
php artisan dte:alert-critical --limit=20 --since-hours=6
```
- Scheduler já executa hourly e grava log em `storage/logs/dte-alertas.log`.
